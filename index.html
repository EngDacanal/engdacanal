☁️ Conexão Supabase
Para salvar dados na nuvem e acessar de qualquer dispositivo, insira as credenciais do seu projeto Supabase abaixo.
Siga o roteiro abaixo para criar o projeto gratuitamente.
📋 Roteiro de Configuração
1️⃣
Criar conta no GitHub
Acesse github.com → Sign up → Use seu e-mail profissional → confirme o e-mail.
🔗 github.com/signup
2️⃣
Criar repositório no GitHub
No GitHub: New repository → Nome: dacanal-sistema → Public → Create. Depois vá em Settings → Pages → Branch: main → /root → Save.
3️⃣
Upload do arquivo HTML
Na página do repositório: Add file → Upload files → Arraste este arquivo HTML → Renomeie para index.html → Commit changes.
4️⃣
Criar conta no Supabase
Acesse supabase.com → Start for free → Continue with GitHub → New project → Nome: dacanal → Region: South America (São Paulo) → Create.
🔗 supabase.com
5️⃣
Criar a tabela de leads no Supabase
No Supabase: SQL Editor → New query → Cole e execute o SQL abaixo:
-- Execute no SQL Editor do Supabase: create table if not exists config (chave text primary key, valor text); insert into config values ('orc_seq','0') on conflict do nothing; create policy "all" on config for all using (true) with check (true); alter table config enable row level security; create table if not exists orcamentos ( id text primary key, ref text unique, seq integer, cliente text, tipo_cliente text, doc text, endereco text, responsavel text, valor numeric, horas numeric, equip integer, complexidade text, data_orc date, status text default 'enviado', obs text, params jsonb default '{}', criado_em timestamptz default now() ); create policy "all" on orcamentos for all using (true) with check (true); alter table orcamentos enable row level security; create table if not exists laudos ( id text primary key, num_laudo text unique, orc_ref text, cliente text, local text, data_laudo date, num_art text, equip integer, valor numeric, status text default 'emitido', obs text, criado_em timestamptz default now() ); create policy "all" on laudos for all using (true) with check (true); alter table laudos enable row level security; -- LEADS (se ainda não criou): create table if not exists leads ( id text primary key, nome text not null, tipo text, cidade text, resp text, tel text, email text, estagio text default 'prosp', valor numeric, equip integer, followup date, origem text, obs text, historico jsonb default '[]', criado_em timestamptz default now(), atualizado_em timestamptz default now() ); alter table leads enable row level security; create policy "public_all" on leads for all using (true) with check (true);
6️⃣
Pegar as credenciais do Supabase
No Supabase: Settings → API → Copie:
• Project URL → cole no campo "Supabase URL" acima
• anon public key → cole no campo "Anon Key" acima
Depois clique em Testar Conexão.
7️⃣
Acessar de qualquer lugar
Sua URL será: seu-usuario.github.io/dacanal-sistema
Salve nos favoritos do celular. Todos os leads ficam no Supabase — acesso de qualquer dispositivo.
